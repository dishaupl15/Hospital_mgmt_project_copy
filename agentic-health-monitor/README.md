# Agentic Health Monitor

A hackathon-ready AI triage assistant that converts free-form patient symptoms into structured clinical insights using a multi-agent LLM workflow and RAG grounding.

## Problem Statement

Healthcare intake is often slow, inconsistent, and difficult to scale. Emergency and primary care workflows need better early triage support that can turn patient symptom text into reliable clinical guidance.

## Solution Overview

Agentic Health Monitor uses a collaborative agent architecture to simulate expert reasoning. The app transforms symptom descriptions into follow-up questions, risk assessments, condition insights, and recommendation summaries.

## Key Features

- 🧠 Multi-agent LLM reasoning for intelligent clinical decision support
- ❓ Dynamic follow-up question generation for symptom clarification
- ⚠️ Risk and urgency scoring with structured recommendations
- 📚 Retrieval-Augmented Generation (RAG) with medical evidence grounding
- 📄 Auto-generated patient health reports
- 💾 Save and review assessment history
- 📤 Export-ready summary reports
- 🔄 End-to-end diagnostic workflow from symptoms to care guidance

## System Architecture

The app is built as a modular layered system:

- **Frontend**: symptom intake, follow-up flow, report display, and history dashboard
- **Backend**: API delivery, multi-agent orchestration, and persistence
- **RAG Layer**: document embedding, vector search, and context retrieval

### Core Modules

- `backend/app/agents/` → specialized AI agents
- `backend/app/core/` → LLM and service configuration
- `backend/app/routes/` → FastAPI endpoints
- `backend/app/rag/` → embedding and retrieval logic
- `frontend/src/pages/` → user workflow and report views

## Installation & Setup

### Backend Setup
```bash
cd agentic-health-monitor/backend
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
copy .env.example .env
```

### Frontend Setup
```bash
cd agentic-health-monitor/frontend
npm install
```

## Environment Variables

Copy `agentic-health-monitor/backend/.env.example` to `agentic-health-monitor/backend/.env` and set:

```env
GROQ_API_KEY=your_groq_api_key_here
SUPABASE_URL=your_supabase_url_here
SUPABASE_KEY=your_supabase_service_role_key_here
VITE_SUPABASE_URL=your_supabase_url_here
VITE_SUPABASE_ANON_KEY=your_supabase_anon_key_here
```

> Keep `SUPABASE_KEY` server-side only. Do not expose it in frontend code.

## How to Run

### Start Backend
```bash
cd agentic-health-monitor/backend
.venv\Scripts\activate
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

### Start Frontend
```bash
cd agentic-health-monitor/frontend
npm run dev
```

Open the UI at: `http://localhost:5173`

## Usage

1. Open the app in the browser
2. Enter patient details and symptom descriptions
3. Review follow-up questions generated by the AI
4. Answer the diagnostic prompts
5. Receive a structured risk assessment and recommendations
6. Save reports and review patient history

## API / Module Summary

### Main Backend Endpoints

- `POST /analyze-symptoms` — analyze symptoms and generate follow-up questions
- `POST /final-assessment` — return final condition, risk, and recommendation
- `POST /save-report` — save completed patient reports
- `GET /history` — retrieve saved reports

### Key Backend Modules

- `agents/orchestrator.py` — coordinates the multi-agent workflow
- `agents/symptom_agent.py` — symptom interpretation
- `agents/clarification_agent.py` — follow-up question generation
- `agents/diagnosis_agent.py` — condition reasoning
- `agents/risk_agent.py` — risk assessment
- `agents/recommendation_agent.py` — recommendation generation

### Frontend Pages

- `src/pages/SymptomForm.jsx` — symptom intake form
- `src/pages/FollowUp.jsx` — follow-up question workflow
- `src/pages/Report.jsx` — report summary and export
- `src/pages/History.jsx` — saved assessment history
